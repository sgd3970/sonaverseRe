# Sonaverse 리뉴얼 – 디자인 시스템 × React 컴포넌트 매핑

> **목적**: Stitch 디자인 시스템 요소를 **실제 React 컴포넌트 이름**으로 고정해, 구현 시 혼선을 줄이고 재사용성을 극대화하기 위함  
> **전제**: 색상·타이포·레이아웃은 `STITCH_DESIGN_SYSTEM_UNIFIED` 기준

---

## 1. 버튼 (Buttons)

| 디자인 역할                   | React 컴포넌트 이름        | 설명 |
|------------------------------|----------------------------|------|
| Primary CTA 버튼             | `SharedButtonPrimary`      | 메인 액션 (예: 문의하기, 자세히 보기) |
| Secondary 버튼               | `SharedButtonSecondary`    | 보조 액션 (예: 목록으로, 취소) |
| Ghost / Text 버튼            | `SharedButtonGhost`        | 배경 없는 텍스트형 버튼 |
| Icon + Text 조합 버튼        | `SharedIconButton`         | 아이콘과 텍스트가 함께 있는 버튼 |
| Icon-only 버튼               | `SharedIconButtonMinimal`  | 아이콘만 있는 버튼 (예: 닫기, 더보기) |

---

## 2. 카드 (Cards)

| 디자인 요소                             | React 컴포넌트 이름             | 사용 위치 |
|----------------------------------------|----------------------------------|----------|
| 기본 컨텐츠 카드 (제목+설명+CTA)       | `ContentCard`                    | 홈, 섹션 리스트 |
| 언론보도 카드 (썸네일+언론사+날짜)     | `PressCard`                      | `/press`, 홈 섹션 |
| 스토리 카드 (썸네일+카테고리+제목)     | `StoryCard`                      | `/sonaverse-story`, 홈 섹션 |
| 제품 카드 (이미지+제품명+태그)         | `ProductCard`                    | `/products/**`, 홈 섹션 |
| 관리자 리스트용 카드/행                 | `AdminListRow`                   | `/admin/**` 리스트 뷰 |

---

## 3. 섹션 & 레이아웃 (Sections & Layout)

| 디자인 요소                            | React 컴포넌트 이름          | 설명 |
|---------------------------------------|------------------------------|------|
| 섹션 타이틀 + 서브타이틀              | `SectionHeader`              | 모든 페이지 공용 섹션 헤더 |
| 섹션 래퍼 (padding, max-width 등)     | `SectionContainer`           | 레이아웃 일관성 유지용 |
| 그리드 레이아웃 (카드 리스트)         | `CardGrid`                   | 스토리/언론보도/제품 등 |
| 2열 레이아웃 (이미지 + 텍스트)        | `SplitSection`               | 제품 소개, 스토리 섹션 등 |
| 페이지 상단 Hero 섹션                 | `PageHero`                   | 홈, 제품 상세, 리스트 상단 등 |

---

## 4. 입력 & 폼 (Inputs & Forms)

| 디자인 요소                       | React 컴포넌트 이름       | 설명 |
|----------------------------------|---------------------------|------|
| 기본 텍스트 입력                 | `FormInput`               | label + input + 에러 메시지 포함 |
| 텍스트 영역                      | `FormTextarea`            | 문의 내용 등 긴 텍스트 입력 |
| 셀렉트 박스                      | `FormSelect`              | 문의 유형, 카테고리 선택 등 |
| 체크박스                         | `FormCheckbox`            | 개인정보 동의 등 |
| 라디오 그룹                      | `FormRadioGroup`          | 단일 선택 |
| 파일 업로드 입력                 | `FormFileInput`           | 문의 첨부파일, 관리자 업로드 등 |
| 폼 섹션 그룹 / 필드셋            | `FormFieldset`            | 구분이 필요한 입력 그룹 |

> 이들 컴포넌트는 `react-hook-form`과 자연스럽게 연결될 수 있도록 `ref` 전달, `name` prop 등 일관된 인터페이스를 갖도록 설계한다.

## 6. 메시지 & 피드백 (Feedback)

|--------------------------|-------------------------|------|

---

## 8. 적용 가이드

1. 새로운 UI를 설계/구현할 때, **먼저 이 매핑표에서 적합한 컴포넌트가 있는지 확인**한다.
2. 완전히 새로운 패턴이라면:
   - 이름을 이 표의 규칙에 맞게 정의한 후
   - `shared/components/**` 또는 `features/**/components`에 추가한다.
3. 컴포넌트 이름은 **디자인 시스템 문서와 맞춰서** 변경/추가하고, 이 문서도 함께 업데이트한다.
