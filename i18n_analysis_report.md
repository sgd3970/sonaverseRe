# 국제화(i18n) 분석 보고서

## 개요
이 프로젝트는 `src/lib/i18n` (클라이언트 컨텍스트 + 서버 딕셔너리)와 `next-i18next` 설정 파일(일부 미사용 추정)을 결합한 커스텀 i18n 구현을 사용하고 있습니다. 현재 App Router 아키텍처는 URL 기반 라우팅(예: `/[lang]/...`)을 사용하지 않고, `localStorage`에 저장된 상태를 기반으로 클라이언트 사이드 렌더링(CSR) 컨텍스트 전환에 의존하고 있습니다.

## 주요 발견 사항

### 1. 컴포넌트 내 하드코딩된 콘텐츠
일부 컴포넌트가 번역 함수로 감싸지지 않은 하드코딩된 한국어 텍스트 배열이나 객체를 포함하고 있어, 로케일이 변경되어도 내용이 변하지 않는 문제가 있습니다.

*   **`src/features/home/components/ProblemSection.tsx`**
    *   **문제:** 기능 목록(안전성, 인체공학 등)이 컴포넌트 내부의 하드코딩된 객체 배열로 되어 있습니다.
    *   **영향:** 선택된 로케일과 관계없이 항상 한국어로 표시됩니다.
    *   **해결:** 데이터를 `public/locales/{locale}/home.json`으로 이동하고 키를 통해 매핑해야 합니다.

*   **`src/features/home/components/StoryHighlight.tsx`**
    *   **문제:** `stories` 배열이 한국어 제목과 요약을 포함한 하드코딩된 모의 데이터(mock data)입니다.
    *   **영향:** 모의 데이터가 번역되지 않습니다.
    *   **해결:** 모의 데이터를 `home.json`으로 이동하거나 API 연동을 구현해야 합니다(현재는 "TODO" 상태). 당장은 JSON으로 이동하는 것이 적절합니다.

### 2. 과도한 하드코딩 폴백 사용
컴포넌트에서 `t('key') || '기본 한국어 텍스트'` 패턴이 광범위하게 사용되고 있습니다.
*   **문제:** 이는 크래시를 방지하지만, 유지보수 부담(코드 내 텍스트 로직)을 가중시키고 번역 키 누락을 감추는 원인이 될 수 있습니다.
*   **권장사항:** `t` 함수가 키(또는 설정된 특정 폴백)를 반환하도록 하고, 모든 키가 JSON에 존재하는지 확인하는 방식에 의존해야 합니다.

### 3. 서버 사이드 기본 로케일의 한계
*   **위치:** `src/app/layout.tsx`
*   **문제:** `RootLayout`이 명시적으로 `getDictionary("ko")`를 호출하고 `<html lang="ko">`를 설정합니다.
*   **영향:** 초기 서버 HTML 페이로드는 *항상* 한국어입니다. 사용자가 영어를 선호하더라도 먼저 한국어 HTML을 받은 후, 클라이언트 사이드 `LanguageProvider`가 하이드레이션되면서 영어로 전환되므로, 콘텐츠 깜빡임(Flash of Content)이나 하이드레이션 불일치가 발생할 수 있습니다.
*   **권장사항:** 장기적으로는 서버가 로케일을 알 수 있도록 URL 기반 라우팅(예: `/[lang]/page.tsx`)으로 마이그레이션해야 합니다. 현재로서는 이 아키텍처의 알려진 한계점입니다.

### 4. 메타데이터 하드코딩
*   **위치:** `src/app/layout.tsx`, `src/app/page.tsx`
*   **문제:** SEO 메타데이터(제목, 설명, OG 태그)가 한국어로 하드코딩되어 있습니다.
*   **영향:** 검색 엔진이 사이트를 한국어 전용으로 색인할 가능성이 높습니다.
*   **해결:** 이를 제대로 수정하려면 서버 사이드 로케일 감지(URL 라우팅)가 필요합니다.

## 조치 계획

1.  **`ProblemSection.tsx` 리팩토링**: 하드코딩된 기능 목록을 `home.json`으로 추출합니다. (완료)
2.  **`StoryHighlight.tsx` 리팩토링**: 모의 스토리 데이터를 `home.json`으로 추출합니다. (완료)
3.  **JSON 확인 및 동기화**: 추출된 키가 `ko/home.json`과 `en/home.json` 모두에 존재하는지 확인합니다. (완료)
